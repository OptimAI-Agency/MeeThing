```typescript
export const resetPasswordSchema = z.object({
  password: z.string()
    .min(8, { message: "Password must be at least 8 characters" })
    .max(72, { message: "Password must be less than 72 characters" })
    .regex(/[a-z]/, { message: "Password must contain a lowercase letter" })
    .regex(/[A-Z]/, { message: "Password must contain an uppercase letter" })
    .regex(/[0-9]/, { message: "Password must contain a number" }),
  confirmPassword: z.string(),
}).refine((data) => data.password === data.confirmPassword, {
  message: "Passwords don't match",
  path: ['confirmPassword'],
});
```

This reuses the same password rules as `signUpSchema` plus adds `confirmPassword` with a `refine` for match validation.

  - `isRecoveryMode` (boolean, false)
  - `password` (string, '')
  - `confirmPassword` (string, '')
  - `loading` (boolean, false)

- Recovery mode detection (`useEffect` on mount):
  ```typescript
  useEffect(() => {
    // Check URL hash for recovery token (supabase-js auto-parses this)
    const hash = window.location.hash;
    if (hash.includes('type=recovery')) {
      setIsRecoveryMode(true);
    }

    // Also listen for PASSWORD_RECOVERY event from supabase-js
    const { data: { subscription } } = supabase.auth.onAuthStateChange((event) => {
      if (event === 'PASSWORD_RECOVERY') {
        setIsRecoveryMode(true);
      }
    });

    // If neither detection fires after 2s, redirect to forgot-password
    const timeout = setTimeout(() => {
      // Use a ref or check current state -- but since this runs once,
      // we need to use a mutable ref to track if recovery was detected
    }, 2000);

    return () => {
      subscription.unsubscribe();
      clearTimeout(timeout);
    };
  }, []);
  ```

  IMPORTANT: The timeout redirect has a closure issue -- `isRecoveryMode` inside the timeout will always be `false` (initial value). Use a `useRef<boolean>(false)` called `recoveryDetectedRef`. Set `recoveryDetectedRef.current = true` alongside `setIsRecoveryMode(true)` in both detection paths. In the timeout callback, check `if (!recoveryDetectedRef.current) { navigate('/forgot-password'); }`.
