With this new validation (placed after the error check block, before `if (!code || ...)`):
```typescript
// CSRF validation: compare callback state to stored state
const expectedState = sessionStorage.getItem("oauth_state");
sessionStorage.removeItem("oauth_state"); // clear immediately, single-use

if (!code || !state || state !== expectedState) {
  toast({
    title: "Connection failed",
    description: "Invalid or expired OAuth state. Please try connecting again.",
    variant: "destructive",
  });
  navigate("/calendar");
  return;
}
```

Key details:
- `sessionStorage.removeItem("oauth_state")` is called BEFORE the comparison, ensuring the token is single-use regardless of outcome
- The check requires both `code` AND `state` to be present, AND `state` to match `expectedState`
- If `expectedState` is null (e.g., user opened callback URL in a new tab), the check fails — this is correct and intentional
- Show a destructive toast so the user knows what happened (per Claude's discretion from CONTEXT.md)

Do NOT change any other code — the session check, edge function invocations, error handling, and JSX template all stay the same.
