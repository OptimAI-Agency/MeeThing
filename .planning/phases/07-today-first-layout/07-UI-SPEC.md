| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, toggle group internal gap |
| md | 16px | Default element spacing, card padding on mobile |
| lg | 24px | Section padding, card padding on desktop |

## Interaction Contracts

### ViewToggle Behavior

| Interaction | Behavior |
|-------------|----------|
| Page load (no `?view` param) | Defaults to today view; URL stays clean (`/calendar`) |
| Page load with `?view=week` | Renders week view; URL shows `?view=week` |
| Page load with `?view=garbage` | Defaults to today view (whitelist validation) |
| Click "Week" toggle | URL updates to `?view=week` via `replace: true`; meeting list shows full 7-day window |
| Click "Today" toggle | URL `?view` param is deleted (clean URL); meeting list filters to today only |
| Browser back/forward | View mode matches URL state; no flash or re-fetch |
| Page refresh | View mode persists from URL; no reset to default |
| Link sharing | Shared URL with `?view=week` opens week view for recipient |

### Scroll Position

| Interaction | Behavior |
|-------------|----------|
| Toggle today to week | Scroll position preserved; no jump to top |
| Toggle week to today | Scroll position preserved; if scroll exceeds new content height, scroll to bottom of content |

Implementation: Single MeetingsList instance receives filtered data as prop (not conditional branches). Save `window.scrollY` before toggle, restore via `requestAnimationFrame` after render.

### Layout Stability (CLS Prevention)

| Constraint | Implementation |
|------------|----------------|
| Content area minimum height | `min-h-[400px]` on the glass-panel content container to prevent collapse when switching from many meetings (week) to few (today) |
| No re-mount animation on toggle | `animate-scale-in` class applied only on initial CalendarHub mount, not on view toggle changes |
| No layout shift on empty state | Empty state container uses same `min-h-[400px]` as populated state |

## ViewToggle Visual Specification

The ViewToggle is the primary new focal element in this phase -- it is the entry point for view switching and draws the eye first after this change. It renders as a compact pill-shaped toggle group inside the overview tab, positioned between the sync button row and the meeting list.

### Layout

```
[Sync icon button ................ right-aligned]
[  Today  |  This week  ]  <-- ViewToggle (primary focal point), center-aligned
[Meeting list / empty state below]
```

### Styling

| Property | Value | Source |
|----------|-------|--------|
| Container | `glass-light rounded-2xl p-1` | Lighter glass than the parent panel for visual nesting |
| Toggle item (inactive) | `text-gray-600 hover:text-gray-900 hover:bg-white/40 rounded-xl px-4 py-2 text-sm font-normal` | Matches existing tab ghost pattern |
| Toggle item (active) | `bg-white text-gray-900 shadow-sm rounded-xl px-4 py-2 text-sm font-normal` | Matches existing active tab pattern (white bg + shadow); secondary surface, not accent |
| Min touch target | `min-h-[36px]` per item | Compact but accessible; parent container padding brings total to 44px tap zone |
| Transition | `spring-smooth` (existing utility: `all 0.3s cubic-bezier(0.25, 0.46, 0.45, 0.94)`) | Matches existing UI motion |
| Alignment | Centered in the overview area, below sync button row | Visually anchored to content, not header |

### States

| State | Visual |
|-------|--------|
| Today active (default) | "Today" has white bg + shadow; "This week" is ghost |
| Week active | "This week" has white bg + shadow; "Today" is ghost |
| Focus visible | `ring-2 ring-primary/20 ring-offset-2` on the focused item |
| Hover (inactive item) | `bg-white/40` background tint |
